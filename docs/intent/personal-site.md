# Personal Site — Confirmed Intent

_Captured 2026-06-19 via interview._

- **Outcome:** A clean, minimal personal site on GitHub Pages showcasing coding projects and (future) publications, easy to extend by editing a data file.
- **User:** Recruiters / hiring managers evaluating Rob — primary purpose is job-seeking, across both SWE/ML-eng and research-leaning roles (no single focus yet).
- **Why now:** A single professional home for his work that he can point people to, and grow over the years.
- **Success:** Live on GitHub Pages, looks professional, real projects written up well, and adding a new project/publication later means editing one structured data file.
- **Constraint:** Content not yet written — project writeups drafted from GitHub repos; publications added over time. Must be genuinely low-maintenance.
- **Tech:** Jekyll (GitHub Pages native, zero build tooling) with projects/publications as YAML data entries. Sections: About, Projects, Publications (with friendly empty state), Experience, Contact, résumé link.
- **Out of scope (v1):** Blog/writing section, custom domain (github.io for now), dark mode, contact forms/backends, analytics.
